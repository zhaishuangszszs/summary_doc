cpp -v
在 Linux 下，使用 cpp -v 命令可以查看 C/C++ 预处理器的版本信息和详细输出。
cpp 命令是 C/C++ 的预处理器，它在编译过程之前对源代码进行宏展开、条件编译和包含文件等预处理操作。
g++ -v -E -x c++ 



在 Linux 下，C++ 静态链接库和动态链接库的默认搜索路径主要包括以下几个：

/usr/lib：这是系统级别的库目录，包含许多标准的系统库。

/usr/local/lib：这是本地库目录，通常用于用户自己安装的软件和库。

/lib：这是系统级别的静态库目录，存放一些基本的系统库。

/lib64：这是系统级别的静态库目录，专门用于 64 位系统架构。

/usr/lib64：这是系统级别的库目录，专门用于 64 位系统架构。

环境变量 LD_LIBRARY_PATH 指定的路径：LD_LIBRARY_PATH 是一个环境变量，用于指定额外的动态链接库搜索路径。可以通过设置该环境变量来指定其他目录。

注意：以上路径仅为常见的默认搜索路径，实际上，系统中的默认搜索路径可能因发行版、系统配置和用户设置而有所不同。

您可以使用以下命令来查看系统的默认库搜索路径：


echo $LD_LIBRARY_PATH
这将显示当前设置的 LD_LIBRARY_PATH 环境变量的值，其中包含的路径是动态链接库的搜索路径之一。

另外，可以使用以下命令查看默认的静态库搜索路径：


gcc -print-search-dirs | grep libraries
这将显示 GCC 编译器的默认库搜索路径，包括静态库的搜索路径。

总结起来，在 Linux 下，C++ 静态链接库和动态链接库的默认搜索路径主要涵盖系统级别的库目录和一些用户级别的库目录，以及通过 LD_LIBRARY_PATH 环境变量指定的其他路径。
